## 3. 프로젝트 개요

🔹 프로젝트 배경

현재 사내 인프라 지원 요청이 메신저, 이메일, 전화, 구두 전달 등 여러 채널로 이루어져 업무 효율성이 저하되고 있음.

요청 단일화: 모든 이슈를 한눈에 파악하고, 처리 프로세스를 자동화하여 업무 효율 향상

요청 추적 관리: 누락 방지 및 반복 문의 감소를 위해 체계적인 티켓 시스템 도입

진행상황 및 통계 제공: 실시간 상태 확인 및 이력 관리, 통계 분석을 통해 업무 프로세스 개선

🔹 프로젝트 목적

통합 요청 관리: 하드웨어, 소프트웨어, 네트워크, 계정 관리 등 모든 요청을 한 시스템에서 접수·관리·모니터링

투명성 및 책임성 강화: 담당자 지정 및 실시간 진행 상황 공유로 책임 소재 명확화

데이터 기반 개선: 티켓 데이터를 분석하여 빈번한 문제 파악 및 향후 인프라 운영 최적화

## 4. 프로젝트 주요 기능

- ✅ 1. 요청(티켓) 관리 시스템
  통합 요청 접수: 하드웨어, 소프트웨어, 네트워크, 계정 관리 등 모든 지원 요청을 하나의 시스템에서 접수
  티켓 자동 분류: 요청 유형에 따라 자동으로 담당자 및 우선순위 지정
  티켓 상태 관리: 접수됨 → 진행 중 → 완료 등 실시간 상태 업데이트
  첨부 파일 지원: 요청 시 관련 파일 업로드 가능

- ✅ 2. 실시간 진행상황 모니터링
  요청자 & 담당자 대시보드: 사용자는 본인의 요청 상태를 확인, 담당자는 할당된 티켓을 한눈에 파악
  실시간 알림: 요청 처리 상태 변경 시 이메일 또는 푸시 알림 전송
  업무 진행 기록: 요청 처리 과정에서의 모든 변경 사항과 커뮤니케이션 로그 저장

- ✅ 3. 티켓 커뮤니케이션 및 협업 기능
  티켓 내 댓글 기능: 담당자와 요청자가 티켓 내에서 직접 소통 가능
  서브태스크(Subtask) 생성: 복잡한 요청을 작은 단위로 분리하여 개별 처리 가능
  티켓 템플릿: 반복적인 요청 유형을 템플릿으로 저장하여 빠른 생성 지원

- ✅ 4. 통계 및 보고 기능
  요청 처리 이력 및 통계: 티켓 처리 속도, 재발율, 해결율 등 주요 지표 분석
  이슈 패턴 분석: 자주 발생하는 문제를 식별하여 사전 예방 조치 가능
  월별/분기별 리포트: 관리자가 요청 처리 현황을 데이터로 확인하고, 의사결정 지원

- ✅ 5. 인증 및 보안 관리
  사용자 역할 기반 접근 제어 (RBAC): 일반 사용자, 관리자, 담당자 등 권한별 기능 제한
  JWT 기반 인증: 보안 강화를 위한 JWT(Json Web Token) 기반 인증 시스템 적용
  로그 기록 및 감사 기능: 티켓 변경 및 주요 활동 내역을 기록하여 보안 감사 가능

## 7. 프로젝트 회고

- ### 아쉬웠던 점

- **전반적인 시간관리 부족**
    - 프로젝트의 전반적인 개발 계획을 세우고 초기 세팅부터 코드 프리즈 이후 QA까지의 과정을 체계적으로 설계한 후에는 비교적 원활하게 진행되었지만, 초반에는 방향을 잡는 데 시간이 걸렸습니다. 특히, 초기 단계에서 세부적인 설계가 부족했던 부분이 개발 진행 중 예상치 못한 문제로 이어졌고, 그로 인해 시간을 낭비하는 경우가 있었습니다. 결국, 시간이 부족하여 초기에 구상했던 기능들을 모두 구현하지 못한 점이 가장 아쉬운 부분으로 남았습니다.
- **우선순위 미흡**
    - 작업의 우선순위를 명확히 설정하지 못해 일정 후반부에 작업이 몰리는 문제가 발생하였습니다. 이에 따라 코드 품질을 충분히 고려하지 못하고 급하게 개발하는 상황이 종종 발생하였습니다.
- **생소한 시스템(TMS) 적용으로 인한 어려움**
    - 기획 단계에서 TMS에 대한 개념이 익숙하지 않아 많은 논의가 필요했습니다. 하지만 이러한 논의를 통해 프로젝트 방향성이 명확해졌으며, 결과적으로 필요한 기능을 효과적으로 반영할 수 있었습니다.
- **API 연동 과정에서의 시간 부족**
    - API 연결이 예상보다 늦어져 프론트엔드 개발에서 일부 기능을 임시 데이터로 대체해 작업해야 하는 경우가 있었다. 촉박한 일정 속에서 코드의 유지보수성과 확장성을 충분히 고려하기 어려운 점이 아쉬웠습니다.

- ### 잘했던 점

- **기획 단계에서의 심층적인 논의**
    - TMS 시스템이라는 생소한 분야임에도 불구하고 많은 논의를 거쳐 필요한 기능들을 세밀하게 정의하였습니다. 이러한 기획 덕분에 개발 단계에서 “이 기능을 추가하길 잘했다”는 생각이 드는 경우가 많았습니다.
- **책임감을 가지고 프로젝트를 진행**
    - 각자의 역할을 책임감 있게 수행하며, 정해진 기간 내에 목표를 달성한 점은 긍정적으로 평가할 수 있습니다. 개발 과정이 힘들고 피로가 누적되는 상황에서도 팀원들은 맡은 일을 소홀히 하지 않았고, 서로의 업무를 미루거나 떠넘기는 일 없이 끝까지 책임감 있게 진행했습니다. 이러한 협업 덕분에 프로젝트를 성공적으로 마무리할 수 있었습니다.
    - 팀원 모두가 적극적으로 개발에 참여하며 각자의 역할을 충실히 수행하였습니다. 특히 서로 다른 스타일의 코드를 접하고 조율하는 과정에서 많은 것을 배울 수 있었습니다. 프로젝트를 진행하며 다양한 기술을 학습하고 적용할 수 있었습니다.
- **멘탈 관리**
    - 멘토링을 받으면서 심각한 상황이라는 것을 인지하고, 짧은 시간 안에 모든 api를 연결해야 하는 상황에서 팀원 모두가 끝까지 포기하지 않고 마감 시간 1초 전 까지  열심히 노력해준 덕분에 생각했던 완성도보다 더 높은 완성도로 기능 개발을 마치게 되었던 거 같습니다. 잠도 조금밖에 못 자고 지치고 피곤한 상태에서도 다들 멘탈 관리를 잘해줬던 거 같습니다.
- **코드 리뷰**
    - 팀원들끼리 서로 개발한 부분에 대해서 코드 리뷰를 진행하면서 코드의 질을 높일 수 있었습니다. 메인 브랜치로 올리기 전에 꼭 팀원들이 코드 리뷰 및 승인을 해야 했기 때문에 서로 알고 있는 정보나 더 효율적으로 코드를 짤 수 있는 방법도 공유했습니다. 또한 라이브러리 설치나 공통 컴포넌트 수정 관련해서는 팀원들이 인지할 수 있도록 PR 참고사항에 설명을 적어서 팀원들간의 코드 충돌을 최소화 시킬 수 있었습니다.

- ### 느낀점

- **협업에서의 소통 중요성**
    - 처음 적용하는 시스템과 생소한 개념이 많았지만, 적극적인 의견 교환을 통해 문제를 해결해 나갈 수 있었습니다. 특히 기획 단계에서의 충분한 논의가 개발 과정에서의 시행착오를 줄이는 데 큰 도움이 되었습니다.
- **테스트 및 QA의 중요성**
    - 개발이 완료된 이후에도 충분한 테스트와 QA를 거쳐야 UI/UX적으로 완성도 높은 시스템이 될 수 있음을 깨달았습니다. 남은 기간 동안 QA를 철저히 진행하여 사용자 경험을 더욱 개선하고자 합니다.
- **트러블 슈팅의 중요성**
    - 이번 프로젝트를 진행하면서 트러블슈팅의 중요성을 팀 전체가 깊이 체감하게 되었습니다. 문제 해결을 위해서는 단순히 코드 수정을 넘어서, 다양한 도구들을 활용하여 문제를 분석하고, 백엔드와 프론트엔드의 상호작용을 면밀히 살펴보는 과정이 필요하다는 것을 깨달았습니다. 특히, 문제 해결 과정에서의 끈기와 세밀한 확인이 얼마나 중요한 역할을 하는지를 실감할 수 있었습니다. 이번 경험을 통해 트러블슈팅을 단순히 오류를 수정하는 과정이 아니라, 시스템 전반을 이해하고 개선할 수 있는 기회로 활용해야 한다는 점을 배웠습니다. 각자가 문제를 분석하고 해결하는 데 필요한 기술을 키우며, 문제가 발생했을 때 빠르게 원인을 분석하고, 체계적으로 해결하는 능력이 프로젝트를 성공적으로 이끌어가는 데 중요한 자산이 된다는 것을 공감하게 되었습니다. 이를 바탕으로 향후 프로젝트에서도 더욱 효과적으로 문제를 해결하고 시스템을 개선해 나갈 수 있을 것입니다.

- ### 개선할 수 있는 점

- **테스트 일정 명확화 및 분배 -** 남은 기간 동안 QA 일정을 구체적으로 정리하고, 우선순위에 따라 테스트를 분배하여 진행할 예정입니다.
- **기능별 테스트 케이스 문서화** - notion 또는 테스트 관리 도구를 활용하여 기능별 테스트 케이스를 정리하고, API 응답값 검증, UI 상태 변화, 오류 메세지 처리 등 체계적인 테스트 기준을 설정하여 누락을 방지할 것입니다.
- **UI/UX 테스트 강화** - 팀원들과 직접 사용자의 입장에서 테스트하여 피드백을 확보할 것입니다. 주요 기능들을 수행하면서, 사용성이 떨어지는 부분, UI가 직관적이지 않은 부분을 개선해나갈 예정입니다. 또한 사용자 경험에 영향을 주는 요소들인 버튼, 모달, 알림 메시지 등의 일관성을 점검할 것입니다.
- **즉각적인 데이터 갱신** - 남은 기간 동안 데이터 상태 관리 관련해서 오류를 수정할 예정입니다. 화면에서 즉각적으로 데이터가 갱신되지 않는 오류가 종종 발생하고 있는데 이 부분 관련해서 react-query를 활용해서 사용자가 서비스를 더 편하게 이용할 수 있도록 오류를 해결해나갈 예정입니다.
